## [Перепрошивка ESP32 через OTA](#) 

> OTA-обновление (от англ. Over-The-Air update, «обновление по воздуху») - способ обновления встроенных систем, при котором обновления передаются через беспроводные сети. Это позволяет распространять новые версии программного обеспечения, снижать затраты на их доставку и ускорять внедрение

Для реализации обновления прошивки ESP32 через OTA необходимо: 

- настроить веб-сервер, который обслуживает файл обновления встроенного ПО;

- настроить ESP32 для подключения к этому серверу и загрузки файла обновления;

- использовать библиотеку для выполнения обновления, например ArduinoOTA;

- добавить код, поддерживающий беспроводную загрузку, в прошивку и загрузить его на микроконтроллер через проводное USB-соединение. Этот код позволяет в будущем подключаться и загружать код по Wi-Fi;

- убедиться, что в обновлённой прошивке остаётся код, поддерживающий OTA, иначе беспроводные обновления не будут возможны в следующий раз. 

Алгоритм процесса обновления прошивки через OTA: 

- устройство загружает новую прошивку порциями и сохраняет её в отдельной области памяти («пространство OTA»);
- как только вся прошивка будет загружена и проверена, загрузчик устройства начнёт действовать. Он переместит новый код из пространства OTA в пространство приложения, а затем передаст управление коду в пространстве приложения.
 
Важно: во время обновления лучше отключить любые подключённые устройства, если они выполняют критичные задачи. По возможности обновлять устройство через защищённое соединение (HTTPS), особенно если обновление выполняется через интернет. 

Пример кода для настройки OTA-обновлений на ESP32 с использованием библиотеки ArduinoOTA: 

- включить библиотеки ArduinoOTA и WiFi в верхнюю часть скетча: 

```
#include <ArduinoOTA.h> 
#include <WiFi.h>
```

- настроить соединение Wi-Fi: вызвать функцию WiFi.begin() и передать имя и пароль сети Wi-Fi;

- инициализировать библиотеку ArduinoOTA: вызвать функцию ArduinoOTA.begin();

- установить имя хоста и пароль для OTA-обновления с помощью функций ArduinoOTA.setHostname() и ArduinoOTA.setPassword();

- настроить функцию для обработки входящих запросов на обновление OTA с помощью функции ArduinoOTA.onStart();

- настроить функцию для обработки хода обновления OTA с помощью функции ArduinoOTA.onProgress();

- настроить функцию для обработки завершения OTA-обновления с помощью функции ArduinoOTA.onEnd();

- вызвать функцию ArduinoOTA.handle() в основном цикле программы, чтобы прослушивать и обрабатывать запросы на обновление OTA.
